## Ответы на контрольные вопросы

Ниже приведены ответы на контрольные вопросы, сформированные на основе теоретического материала:

**1. Какие основные компоненты входят в структуру Android-проекта?**

Ключевые элементы структуры проекта в Android Studio включают:
- **`Manifests`** — хранит файл `AndroidManifest.xml`, который содержит описание приложения.
- **`Java` / `Kotlin`** — директория для исходного кода (активности, классы, бизнес-логика).
- **`Res` (Resources)** — папка ресурсов, включающая макеты (`layout`), графику (`drawable`), строковые значения (`values/strings.xml`), стили и цвета.
- **`Gradle Scripts`** — скрипты сборки (например, `build.gradle`), отвечающие за зависимости и настройки компиляции.

**2. Для чего нужен файл AndroidManifest.xml?**  
Файл `AndroidManifest.xml` служит для декларирования параметров приложения. Он предоставляет операционной системе необходимые сведения: название, иконку, перечень активностей (экранов) и требуемые разрешения на доступ к функциям устройства.

**3. Чем отличается minSdkVersion от targetSdkVersion?**  
- **`minSdkVersion`** определяет минимальную версию Android, необходимую для установки и запуска приложения.
- **`targetSdkVersion`** указывает версию Android, для которой приложение было разработано и оптимизировано. На более новых версиях ОС приложение также работает, но могут применяться механизмы совместимости, если версия устройства ниже целевой.

**4. Что такое AVD и для чего он используется?**  
**AVD** (Android Virtual Device) представляет собой конфигурацию виртуального устройства (эмулятора). Оно позволяет запускать и отлаживать приложения непосредственно на компьютере, eliminating необходимость в использовании физического смартфона или планшета.

**5. Как изменить текст приложения без изменения кода активности?**  
Для изменения текста без правки кода следует использовать систему ресурсов. Строки выносятся в файл **`strings.xml`** (расположен в `res/values`). В разметке (`layout`) или коде программы используются ссылки на эти ресурсы (вид `@string/имя_строки`). Это позволяет менять текст приложения путем редактирования исключительно файла `strings.xml`.

## Вывод

В результате выполнения лабораторной работы было создано приложение MyFirstApp.

В процессе работы была изучена архитектура Android-проекта в среде Android Studio, включая назначение ключевых элементов: манифеста `AndroidManifest.xml`, директории ресурсов `res` и скриптов сборки Gradle. Также было настроено и создано виртуальное устройство (AVD) для отладки.

Основная задача заключалась в редактировании пользовательского интерфейса через файл разметки `activity_main.xml`. В макет был добавлен второй элемент `TextView`, скорректированы размеры шрифтов и цвета. Все текстовые строки были вынесены в ресурсный файл `strings.xml`, что обеспечивает возможность изменения текста без вмешательства в код активности.

Финальная версия приложения успешно запущена на эмуляторе AVD, что свидетельствует о корректности реализации и настроек проекта.
